Importing MS Word files into Flare

 

Make sure to clean up your MS Word document before importing into Flare by eliminating inline styles; use the MS Word Styles feature instead to create features like bold and italic. Using heading styles is also a good idea.

Be aware of how your Word document is named. For each Word document that you import, Flare will create a folder in the Content Explorer window and name that folder exactly after the Word document. If you have divided your Word document into topics by using section breaks, all of those topics will live in this folder. Flare names these topics based on the initial text in the topic, unless you create unique file names for these topics.

Flare lets you install MS Word documents even if you don’t have the MS Word application!

Follow these instructions to import an MS Word file.

  1. Open the Project Ribbon.

 

  1. Select the dropdown menu Import. Select MS Word Documents. The Import Microsoft Word Wizard window opens.

 

  1. Select either the Import into a new project or the Import into this project radio button. Select Next.

 

  1. Select the green “+” sign on the lower right corner of the Wizard window. The Open File window opens.

 

Select the Link Generated Files To Source Files checkbox if you would like to maintain the link to your source Word files. This is helpful if you want to edit files in the MS Word app for a while and then re-import them. If you’re ready to let go of Word, deselect this checkbox and edit the files from within the Flare project going forward.

 

  1. Navigate to the Word document that you would like to import and select. The Word document now appears in the list of MS Word Files in the Import Microsoft Word Wizard window.
  1. Repeat Steps 4 and 5 to add more Word files. Use the arrow key to arrange the order. This will affect where the files are placed in the TOC file that is created automatically when your project is loaded. elect Next.  The New Topic Styles window opens; import any MS Word Styles here.
  1. Select Next.  The Options window opens; select options. Select Next. The Stylesheet window opens.

 

In the Stylesheet window you can choose to:

  1. Associate a stylesheet with the project.
  2. Choose to preserve or not to preserve MS Word Styles.
  3. Convert inline formatting to CSS styles.

 

Note: If you choose “Preserve MS Word Styles” your CSS stylesheet will contain a great deal of child elements. If you select “Don’t Preserve MS Word Styles” you will have to recreate your styles in CSS, but you can do it on the parent level. If you have lots of inline formatting, rather than use styles from your Word document, you can click “Convert inline formatting to CSS styles” but you can’t use the Wizard to map those styles and the converted styles will have generic names which will make it hard to figure out which styles do what.

 

  1. Select Next. The Paragraph Style Mapping window opens.

 

Select Next. The Character Style Mapping window opens.

 

  1. Select Finish. The Accept Imported Documents window opens. Review generated documents and either accept or cancel the import.

 

  1. The topic appears in the Content Explorer window.
